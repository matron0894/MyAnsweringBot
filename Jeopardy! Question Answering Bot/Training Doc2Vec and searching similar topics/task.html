Description

You've moved on to the most important part of every machine learning project â€” training a model!

In this stage of the project, train a Doc2Vec model, using the Jeopardy! corpus. The model is the base for the question answering bot. It takes a request and returns some of the closest topic from the preprocessing corpus, which allows finding an answer to the user's question. When training the model, it is essential to choose the parameters correctly. You should use the Doc2Vec documentation to complete the stage. Let's experiment with the parameters!

  • vector_size determines how many features each vector is composed of. The larger the size of the vector, the greater the difference between them;
  • window determines how many adjacent words will be taken into account when creating a vector;
  • min_count allows ignoring rare words;
  • workers influence the learning rate of the model by distributing calculations between different cores;
  • epochs help improve prediction accuracy by training in several iterations. More iterations lead to a better result, but this can slow down the model significantly.

To make sure the model works correctly, you will receive some requests from the corpus. The similarity percentage will show you how correct the model is. If this number is lower than 0.5, try to find different parameters for the model's training. Similarity percentage should be printed in a 100% system. So, for example, if the model gives the similarity rank of 0.92, your program should convert it to 92%.

Objectives

In that stage, your program should:

  1. Preprocess the user's input and the training corpus as before;
  2. Train the Doc2Vec model using the preprocessed corpus;
  3. Feed the preprocessed user's request to the model;
  4. Print a number of the closest topic and similarity percentage using a message I know this question: its number is (the number of the closest topic). I'm (similarity percentage)% sure of this.
