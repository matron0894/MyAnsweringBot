Description

The last stage of the project infers providing the answer to users. Your program should take the number of the closest topic and find a row with the same number in the training data. When the program finds the answer in the answer column of the table, it should print it.

If users want to play with the QA bot further and ask about something else? Your program should give an opportunity to ask a few questions, not only one. For this, the answer search algorithm should be repeated until users want to end the conversation with the question-answering system. When it happens, your program should print a farewell message.

Objectives

In this stage, your program should:

  1. Preprocess the user's input and the training corpus as before and train the model;
  2. Using the number of the closest topic, find the answer in the training data;
  3. Return the answer to the user;
  4. Print Do you want to ask me again? (yes/no) and input the user's response;
  5. If the response is yes, the program should print Ask me something! again and repeat the algorithm: get the request, preprocess it, feed it to the model, print the answer, and ask the user again if is there anything else they want to know. If the response is no, your program should print It was nice to play with you! Goodbye!
Examples

Example 1: the QA bot finds the answer to a request.

Hello! I'm Jane, a question answering bot who knows answers to all the questions from the 'Jeopardy!' game.
Ask me something!
> Spanish for "mask", this cosmetic is used to color the eyelashes.
Let's play!
I know this question: its number is 10569. I'm 78% sure of this.
The answer is mascara.
Do you want to ask me again? (yes/no)
> no
It was nice to play with you! Goodbye!
