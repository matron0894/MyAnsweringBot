Description

Let's take a look inside the question-answering system! In this stage, we will see what happens after the bot takes a user request and how it processes it.

Text preprocessing is a necessary part of every NLP project, including the creation of a question answering bot. In NLP tasks, words are often converted to vectors, so a program can handle them and make some operations. But the Hello, world! and hello world sentences have different vectors because letter case and punctuation marks influence the vectors. To get rid of the difference, you need to preprocess the text.

Write a program that will be used for preprocessing a training corpus and userâ€™s queries. This program should take a string, convert it to lowercase, tokenize it, and remove the punctuation. As different tokenization functions provide different output, use the word_tokenize() function from the NLTK package to pass all tests correctly.

You'll need the training corpus in the following stages. In this project, we will use a sample of the 200,000+ Jeopardy! Questions dataset. You can load it from the jeopardy.json file. Download the sample from here: jeopardy.json.

Objectives

In this stage, your program should:

  1. Read a file that contains a training corpus;

  2. Print the greeting and take the user's input as before;

  3. Preprocess both the training corpus and a user request as follows:

    • Convert the text to lowercase with the lower() function;

    • Tokenize the text by words;

    • Remove the punctuation marks from the token list.

  4. Store the training corpus in a list of lists where each list is a tokenized sentence; it will be useful for the next stage;

  5. Print the Let's play! message and the result of tokenization for the user request.
